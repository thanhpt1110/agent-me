# PA-hybrid experiment reverted

_2026-05-10 · short follow-up session._

## TL;DR

Built the PA / Claude-Code hybrid (MCP_CLI env-var swap) over the
prior marathon session. Came back the next day to validate it
end-to-end, found PA's enterprise-source auth had not survived the
overnight gap (all 8 sources 401), and the user decided the
hypothesised auth-retention win wasn't real. Reverted the 3 PA
commits with `git revert` (no force-push, history preserved). The
brief continues to work claude-only (12 items, 64s for the
pre-revert run; 12 items, 147s post-revert with normal MCP latency
variance).

## Sequence of events

1. Resumed session per the `2026-05-10-session-end-pa-hybrid-ready.md`
   handoff. Plan: validate PA mode end-to-end (the only "next
   session priority #1" item).
2. Pre-flight check on auth state surfaced the killer: `pa auth status`
   showed all enterprise sources (NVBugs, OneDrive, GDrive,
   Confluence, Jira, GitLab, ServiceNow) at 401. The whole point of
   PA was supposed to be longer auth retention than Claude Code (which
   expires daily). One overnight gap was already enough to invalidate
   that.
3. Ran the claude-mode brief as a baseline anyway: 12 Jira items,
   64s, exit 0. Confirmed the claude-only path is healthy.
4. User decision: drop PA, revert. The user would tune the prompt later;
   the prompt-tuning that PA was supposed to enable, the user wants to
   do directly without the env-var indirection.
5. `git revert` (preserves history) of the 3 PA commits, in reverse
   chronological order:
     - `efddcb6` — STATE.md + handoff doc → revert `fd8799d`
     - `bdadca1` — Bridge PA-mode awareness → revert `85c7119`
     - `d8907db` — Brief MCP_CLI env var → revert `2dd8b40`
   No conflicts; clean apply on all three.
6. Sanity-test post-revert brief: 12 Jira items, 147s, exit 0.
   Coverage matches; timing variance is normal MCP latency.
7. Updated STATE.md to reflect the actual claude-only state (the
   reverted STATE.md was from before the daily-brief shipped, so it
   undercounted Done items; rewrote it).

## Why this was the right call

- **The auth-retention hypothesis didn't survive its first
  test.** That was the load-bearing reason to take on the hybrid
  abstraction. With it gone, PA mode is just slower (5–15s cold
  start) for no gain.
- **The abstraction cost is real.** The PA path adds branches in
  `daily_brief.py` and `app.py` that have to be maintained as the
  brief and bridge evolve. Removing it reduces the surface area for
  every future change.
- **Reverting is cheap and reversible.** `git revert` keeps the
  PA commits in history. If the user's prompt tuning hits a wall and
  PA's MCP retry behaviour or different model class becomes
  attractive again, we can cherry-pick the 3 commits back.
